# Plan: Yjs Compatibility & Interoperability Test Suite

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement a language-agnostic cross-compatibility test framework to validate that the Dart `yjs_dart` library and the Go `github.com/reearth/ygo` library are 100% wire-compatible and produce identical document states across sequential, concurrent, out-of-order, duplicated, snapshot, and real-world editing scenarios.

## Architecture

We will structure the suite under `compatibility_test/` at the project root:
```
compatibility_test/
├── cases/                     # Declarative case definitions
│   ├── 01_basic_insert/
│   │   ├── steps.json         # Language-agnostic execution steps
│   │   └── expected.json      # Final state assertions
│   ├── 02_go_to_dart/
│   ├── ...                    # All 15 cases
│   └── fixtures/              # Generated binary update files (.bin)
├── dart_runner/               # Dart test runner interpreter
│   ├── pubspec.yaml           # Depends on yjs_dart
│   └── bin/runner.dart        # Interpreter script
└── go_runner/                 # Go test runner interpreter
    ├── go.mod                 # Depends on github.com/reearth/ygo
    └── main.go                # Interpreter binary
```

### The Fixture Protocol
1. **Mode A: Generator (Dart)**: Runs the case's `steps.json`. When an update or diff is exported, it writes the binary bytes directly to `cases/<case>/fixtures/<id>.bin`. At the end of execution, it writes the final document state to `cases/<case>/expected.json`.
2. **Mode B: Verifier (Go)**: Reads the case's `steps.json`. When `import_update` is requested, it loads the binary bytes from `cases/<case>/fixtures/<id>.bin` (generated by Dart). At the end of execution, it asserts that its local Go YDoc state matches `cases/<case>/expected.json`.
3. **Mode C: Cross-Generator (Go -> Dart)**: Go runs as generator writing to `.bin`, Dart verifies.

## 15 Target Cases to Define

1. **Basic Insert (Dart -> Go)**: Simple insert on `YText`.
2. **Go -> Dart**: Simple insert on Go side, sent to Dart.
3. **Out-of-Order Updates**: Updates applied out-of-order (`u3`, `u1`, `u2`) producing converged result.
4. **Duplicate Updates**: Applying the same update multiple times has no duplicate effects (idempotency).
5. **State Vector**: Generating updates using target state vector diff.
6. **Snapshot**: Taking a snapshot, pruning/deleting, and restoring into a new document.
7. **Anti-interleaving**: Concurrent inserts at the same index converge to a contiguous block.
8. **Concurrent Overlapping Deletes**: Concurrent deletes on overlapping ranges.
9. **Map**: Storing nested maps and arrays, validating JSON representation.
10. **Array**: Appending, inserting, and deleting from a shared array.
11. **Large Document**: Inserting 10,000 characters/lines, comparing SHA256 hashes of results.
12. **Fuzzing**: Executing 500+ randomized operations, verifying SHA256 of text matches.
13. **Persistence**: Saving the full state, restoring, and making subsequent edits.
14. **Incremental Sync**: Only exchanging updates, never full documents.
15. **SupaNotes Case**: Real-world markdown editing (concurrent task check + title edit) convergence.
