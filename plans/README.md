## Dependency notes

- **058** é independente dos planos de editor (046–057): toca sync/projeção/agente. Pode correr em paralelo, exceto se tocar os mesmos arquivos de `note_sync_coordinator` (coordenar com 047/049). Fases P0/P1 do 058 têm prioridade sobre 046+.

- **046 must land first.** It is the characterization-test safety net that
  makes 047 (deletion race), 048 (flush-dispose race), and 049 (diff-covers-
  all-types) safe to ship. The fix plans are in lockstep with these tests:
  their executors flip the `BUG 04X` assertions to assert the FIXED behavior.
- **047, 048, 049 are independent of each other** but all require 046. They
  can run in parallel after 046.
- **050, 051, 054, 055 are independent of the bug-fix sequence** and can run
  in any order, in parallel with 046 if needed (051 touches
  `note_editor.dart` lines that 052 also touches — coordinate; the plan
  spells out the constraint).
- **052 depends on 051 only for line-number stability** — the fix is surgical
  (drop `ref.watch` in `build`); if 051 hasn't landed, take care to insert
  the new code BEFORE the stylesheet cache block, not after.
- **053** (header auto-promotion fix) is independent of the others; can run
  early if desired. Recommended to wait for 046 because the editor rebuild
  path becomes safer to touch when characterization tests are in place.
- **056** (sliver migration) is a UX-only refactor; no behavioral contract
  with the sync loop, so it can ship anytime. The smoke test section is
  critical here because no automated test covers the nested-scroll UX.
- **057** (unify the diff logic) is the stretch plan. STRICT dependency on
  046, 047, 048, 049 — it relocates the very code those plans fix. Without
  them, the unifying refactor will reintroduce the bugs. Schedule this
  LAST.
